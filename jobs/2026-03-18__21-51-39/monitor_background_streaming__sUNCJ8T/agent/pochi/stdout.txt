### Key Features of `solve.py`:
- **Sandbox Management**: Creates a new E2B sandbox using `Sandbox.create()`.
- **Task Info**: Saves the sandbox ID to `/home/user/e2b_task_info.json` on the local machine.
- **Script Creation**: Writes the requested `task.sh` bash script inside the sandbox and makes it executable.
- **Background Execution**: Runs `task.sh` as a background command and waits for it to complete.
- **Stdout Capture**: Captures the stdout from the background command and writes it to `/home/user/captured_stdout.txt` inside the sandbox.
- **Sandbox Persistence**: Ensures the sandbox remains open after the script completes by not calling `sandbox.close()`.

```python
import os
import json
from e2b import Sandbox

def main():
    # 1. Create a new E2B sandbox.
    # Assumes E2B_API_KEY is already set in the environment.
    sandbox = Sandbox.create()
    
    try:
        # 2. Save the created sandbox ID to a local file /home/user/e2b_task_info.json under the key sandbox_id.
        sandbox_id = sandbox.sandbox_id
        task_info = {"sandbox_id": sandbox_id}
        with open("/home/user/e2b_task_info.json", "w") as f:
            json.dump(task_info, f)
        
        # 3. Inside the sandbox, create a bash script at /home/user/task.sh with the exact following content:
        task_sh_content = """#!/bin/bash
echo 'Initializing...'
sleep 1
echo 'Running background job...'
sleep 1
echo 'Job complete.'
"""
        sandbox.files.write("/home/user/task.sh", task_sh_content)
        
        # 4. Make the script executable (chmod +x /home/user/task.sh) inside the sandbox.
        sandbox.commands.run("chmod +x /home/user/task.sh")
        
        # 5. Run the script as a background command using the E2B SDK.
        # sandbox.commands.run(..., background=True) returns a CommandHandle
        handle = sandbox.commands.run("/home/user/task.sh", background=True)
        
        # 6. Monitor or wait for the background command to finish, capturing its stdout.
        # handle.wait() waits for the process to finish and returns a CommandResult
        result = handle.wait()
        captured_stdout = result.stdout
        
        # 7. Write the captured stdout to a file inside the sandbox at /home/user/captured_stdout.txt.
        sandbox.files.write("/home/user/captured_stdout.txt", captured_stdout)
        
        # 8. Do not close or kill the sandbox at the end of the script, so the verification step can check the files.
        # We simply do not call sandbox.close().
        print(f"Successfully created sandbox {sandbox_id} and executed task.")
        
    except Exception as e:
        print(f"An error occurred: {e}")
        # In case of error, we might still want to close the sandbox, but the requirement says 
        # "Do not close or kill the sandbox at the end of the script", so we follow that.
        raise e

if __name__ == "__main__":
    main()
```
