The script performs the following actions:

2.  **Installs `uv`** inside the sandbox using the specified curl command.
3.  **Creates a virtual environment** at `/home/user/venv` within the sandbox using `uv venv`.
4.  **Installs `pandas` and `numpy`** into the virtual environment using `uv pip`.
5.  **Verifies the installation** by printing the `pandas` version and saving it to `/home/user/pandas_version.txt` inside the sandbox.
6.  **Saves the sandbox ID** to a local file at `/home/user/e2b_task_info.json` in the format `{"sandbox_id": "<the_id>"}`.

The script is fully automated and uses the `e2b` Python SDK. It assumes that the `E2B_API_KEY` environment variable is already set.

You can run the script with:
```bash
python setup_env.py
```

The script does not close the sandbox upon completion to allow automated tests to connect to it and verify the environment. The sandbox will be automatically cleaned up after the 600-second timeout.

### Script Content:
```python
import json
import os
from e2b import Sandbox

def setup_environment():
    # 1. Starts a new E2B Sandbox with a timeout of at least 600 seconds.
    sandbox = Sandbox(timeout=600)
    print(f"Sandbox started with ID: {sandbox.id}")

    try:
        # 2. Uses the E2B SDK to run a command that installs `uv` in the sandbox.
        install_uv_cmd = "curl -LsSf https://astral.sh/uv/install.sh | sh"
        sandbox.commands.run(install_uv_cmd)
        
        # Path to uv after installation (default is ~/.cargo/bin/uv)
        uv_path = "~/.cargo/bin/uv"
        
        # 3. Uses `uv` to create a virtual environment at `/home/user/venv` inside the sandbox.
        # Ensure /home/user exists (it should, but just in case)
        sandbox.commands.run("mkdir -p /home/user")
        create_venv_cmd = f"{uv_path} venv /home/user/venv"
        sandbox.commands.run(create_venv_cmd)

        # 4. Uses `uv pip` to install `pandas` and `numpy` into that virtual environment.
        # Use the created venv's python with uv pip
        install_packages_cmd = f"{uv_path} pip install --python /home/user/venv/bin/python pandas numpy"
        sandbox.commands.run(install_packages_cmd)

        # 5. Runs a Python command inside the sandbox using the virtual environment's Python
        # to print the pandas version, and saves the output to /home/user/pandas_version.txt.
        verify_pandas_cmd = (
            "/home/user/venv/bin/python -c 'import pandas; print(pandas.__version__)' "
            "| tee /home/user/pandas_version.txt"
        )
        result = sandbox.commands.run(verify_pandas_cmd)
        print(f"Pandas version in sandbox: {result.stdout.strip()}")

        # 6. Writes the created sandbox ID to a local file at /home/user/e2b_task_info.json
        task_info = {"sandbox_id": sandbox.id}
        with open("/home/user/e2b_task_info.json", "w") as f:
            json.dump(task_info, f)
        print("Sandbox ID written to /home/user/e2b_task_info.json")

    except Exception as e:
        print(f"An error occurred: {e}")
        raise
    finally:
        # We don't close the sandbox here because the automated tests need to connect to it.
        # The timeout will handle the cleanup.
        pass

if __name__ == "__main__":
    setup_environment()
```   
